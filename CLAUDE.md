## Продукт и аудитория
Три направления:
1. **Сеансы** (главная) — индивидуальная работа с телом и внутренним состоянием.
   Три программы на выбор: «Атлант + крестец + спина» (1 сеанс), «Восстановление
   физического здоровья» (4–5 сеансов), «Комплексное восстановление тела и сознания»
   (5–6 сеансов, самая полная — визуально выделена карточка).
2. **Ретрит «Возвращение к себе»** — групповой формат, 3 часа, семь шагов
   (дыхательные практики → альфа-состояние → работа с подсознанием → интеграция).
3. **Обучение** — остео-биодинамический подход, 4 дня/40 часов, для специалистов
   телесных практик (массажисты, остеопаты, кинезиологи, косметологи).

Публичное имя everywhere: **Ержан Кабдрашит (Тусупов)** (в родительном падеже — «Кабдрашита
(Тусупова)»). Исключение — строка под сертификатами (там уже отдельным предложением
поясняется, что Тусупов — рабочее имя, добавлять туда «(Тусупов)» ещё раз излишне) и
юридическая строка футера «ИП Кабдрашит, Республика Казахстан» (название регистрации ИП,
не публичное имя).

## Технический стек
- Три тонких HTML-файла (index.html, retreat.html, training.html), общие `site.css` + `site.js`
- Vanilla JS, без фреймворков; fluid-типографика через clamp()
- Общая шапка/футер повторены в каждом файле (нет server-side includes на GitHub Pages).
  Навигация РАЗНАЯ по замыслу хендоффа: на index.html — якоря на секции самой страницы
  (Метод/Программы/Вопросы, т.к. это одностраничный флоу), на retreat.html/training.html —
  сквозные ссылки между страницами (Сеансы/Ретрит/Обучение, активная подчёркнута золотым
  через класс `.current`)
- Чистые URL без .html: nav/footer/canonical ссылаются на `./`, `./retreat`, `./training`
  (и на будущие `./oferta`, `./privacy`, `./consent`) — GitHub Pages отдаёт файл.html по
  запросу без расширения, но КАНОНИЧЕСКИЙ url должен быть без расширения (см. LANDING-GUIDE)
- Фото (из хендоффа, общие для всех страниц): photos/hero/hero-2.* (=portrait-b-w, hero
  главной), photos/about-founder.* (=hero-w, About главной / Ведущий на Ретрите / hero
  Обучения), photos/video-poster-2.* (=portrait-a-w, постер видео главной / hero Ретрита)
- photos/certs/ (6 сертификатов cert-1..6, WebP+JPG full + thumb) — отдельная секция
  «Дипломы и сертификаты» (id="certs") только на index.html, между блоком «Метод» и
  «С чем приходят». Дизайн по хендоффу design_handoff_certificates/: горизонтальный
  скролл, крупные карточки (frame 360px, object-fit:cover), портрет 238px / ландшафт
  514px, подпись = школа-эйбров + название. Лайтбокс новый (footer-бар «нажмите чтобы
  закрыть», клик по оверлею закрывает). Разметка карточек СТАТИЧНАЯ в index.html
  (не генерится JS) — JS только вешает лайтбокс на .cert[data-full].
  ⚠ Было 7 сканов, стало 6: два дубля Neel (одинаковый диплом) объединили в один.
  У путей картинок ?v=4 — cache-busting; при замене файлов поднять число во всех
  ссылках cert-*.(webp|jpg) в index.html.
- Видео: video/method.mp4, постер — photos/video-poster-2.jpg (только на главной)
- Деплой: GitHub Pages, репозиторий Xenaja/tusupov-method

## Запись — по городам, без общего WhatsApp/TG на всех
- **Казахстан → WhatsApp** (кнопки `data-wa="текст сообщения"`, номер `WA_KZ`)
- **Москва → Telegram-бот** (кнопки `data-tg="код_места_клика"`, `TG_BOT`,
  start-параметр = код места: program_hero_msk, program_dates, retreat_hero_msk,
  training_hero и т.д. — коды говорящие, свои на каждой странице)
- Кнопки записи и «узнать стоимость/даты» — ВСЕГДА пара (Москва + Казахстан) на страницах
  «Сеансы» и «Ретрит». На «Обучении» — ОДНА кнопка (только Telegram-бот, ветка обучения) —
  так задано в тексте, не пара.
- На главной кнопки внутри карточек программ («Узнать стоимость») ведут на `#zapis`
  (скроллят к hero, где стоит пара город-кнопок) — так задумано хендоффом (один button на
  карточку вместо двух), а не отдельный WA/TG на каждую из 3 карточек.
- ⚠ ПЛЕЙСХОЛДЕРЫ: `TG_BOT="PLACEHOLDER_BOT"`, `WA_KZ="70000000000"` — в начале `site.js`,
  одно место на все три страницы. Заменить перед запуском рекламы.
- Отдельной floating sticky-кнопки нет ни на одной странице — её роль играет всегда
  видимый sticky-nav (`position:sticky`) с кнопкой «Записаться»/«Оставить заявку».

## Юридические ограничения (важно!)
НЕЛЬЗЯ использовать слова: «лечение», «исцеление», «диагностика», «доктор»,
«целитель», «реабилитация», обещания «вылечим/избавим/гарантируем результат».
На странице «Обучение» отдельно смягчены две формулировки из исходных материалов:
«снятие боли и исцеление» → результаты для клиентов без мед. обещаний;
«постоянный поток клиентов» → «возможность практиковать сразу после обучения».
Стоимость нигде на сайте не публикуем — только через менеджера.
